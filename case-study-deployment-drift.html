DentSignal engineering case study · CI/CD

Source was current. The live domain was not.

A source-only test could pass while the public surface still served stale content. The repair made that disagreement observable and recoverable.

Area
Vercel, GitHub Actions, custom-domain checks
Evidence
PR-reported historical evidence
Boundary
Cited revisions, not current uptime

01 / Problem

A green source check missed deployment drift.

The repository contained the intended change, but the custom domain could still serve an older build. A local or source-level assertion could not prove what the deployed surface returned.

Root cause: the verification chain stopped before the exact hosting project, deployed URL, and custom domain.

02 / Repair

Connect source, deployment, and domain contracts.

  • Added cache-busted custom-domain smoke checks with explicit stale-phrase rejection.
  • Wired the copy contract into CI so source drift could fail before release.
  • Audited the exact Vercel project without printing credentials or project identifiers.
  • Removed a conflicting build-root override and created an explicit deployment path for the checked-out revision.

03 / Validation

The PRs separate diagnosis from correction.

At the cited revisions, the PR record reports source-contract checks, exact-project deployment verification, deployment-URL checks, and custom-domain checks. The result is historical evidence for that repair sequence, not a promise of perpetual availability.

Before: deployed-page drift could remain silent. After: source and live-domain contracts were both explicit.

04 / Rollback

Return to a known-good revision as one unit.

Rollback means reverting the workflow or configuration change and redeploying the prior known-good frontend revision. Source and hosting configuration must remain aligned.
