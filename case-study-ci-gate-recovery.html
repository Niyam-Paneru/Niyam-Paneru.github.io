DentSignal engineering case study · CI/CD

Eleven red gates were not one bug.

The useful move was classification: independent failures in backend code, frontend contracts, security checks, dependencies, and workflow wiring needed independent causes.

Area
Python, TypeScript, security and supply chain
Evidence
PR-reported historical evidence
Boundary
Eleven gates at one cited revision

01 / Problem

The pipeline failed across unrelated lanes.

The PR records eleven failed gates spanning Ruff, handler registration, CORS, secret scanning, supply-chain scripts, dependencies, and frontend mocks.

Root cause: a single red pipeline hid several contract failures. Treating it as one issue would have encouraged broad, risky changes.

02 / Repair

Fix each contract at its source.

  • Repaired missing imports and types, then registered an orphaned handler.
  • Corrected the rate-limit response and cross-origin contract.
  • Removed key-like documentation text and fixed scanner paths and arguments.
  • Upgraded the bounded vulnerable dependencies and restored frontend API/mock contracts.

03 / Validation

A gate inventory made completion reviewable.

At the cited revision, the merged PR enumerates each failing gate and the corresponding correction. This is PR-reported historical evidence; the checks were not rerun for this portfolio change.

Before: eleven named gates failed. After: the repair merged across the backend, frontend, security, and supply-chain lanes described in the PR.

04 / Rollback

Keep workflow and source contracts together.

Rollback is a unit revert of merge commit 7252c74. The workflow and application corrections should not be separated casually because the restored checks rely on the repaired contracts.

05 / Source

Exact merged change record.

PR 90 / merge commit 7252c74.

This is engineering proof, not customer proof. It does not claim that every current CI gate passes, that the full product is secure, or that a customer outcome followed.
